**Helper `proxycurlFetch<T>(path: string, params: Record<string, string>, schema: ZodSchema<T>): Promise<T>` :**
- Construit l'URL avec `URLSearchParams`
- Header `Authorization: Bearer ${process.env.PROXYCURL_API_KEY}`
- Lance `fetch(url, { headers })`
- Si `!response.ok`, throw avec le status et le body text
- Parse JSON et valide avec `schema.parse(json)`

**Fonction principale `collectParisFreelanceData(): Promise<FreelanceParisEntry[]>` :**
- Guard : si `!process.env.PROXYCURL_API_KEY` throw `'PROXYCURL_API_KEY manquant'`
- Étape 1 — Person Search : `GET /proxycurl/api/v2/search/person` params `{ country: 'FR', city: 'Paris', headline: 'freelance', page_size: '5' }` — log `profiles_found`
- Étape 2 — Pour chaque `linkedin_profile_url` de `results` : `GET /proxycurl/api/v2/linkedin` param `{ linkedin_profile_url: url }` — prendre la première expérience où `ends_at` est `null` ou `undefined` (poste actuel) — log `profile_extracted`
- Étape 3 — Company Profile : si `company_linkedin_profile_url` existe, `GET /proxycurl/api/linkedin/company` param `{ url: companyUrl }` — log `company_page`
- Étape 4a — Employee Count total : `GET /proxycurl/api/linkedin/company/employees/count` param `{ url: companyUrl }`
- Étape 4b — Employee Count freelances : même endpoint avec params `{ url: companyUrl, keyword_regex: 'freelance' }`
- Calcul ratio : `totalEmployees > 0 ? Math.round((freelanceCount / totalEmployees) * 100) : 0`
- Si une étape échoue pour un profil (try/catch individuel), log l'erreur et pousser `{ ...profile, companyStats: null }`
- log `company_analyzed` avec companyName, totalEmployees, freelanceCount, freelanceRatio
- Retourner le tableau `results`
