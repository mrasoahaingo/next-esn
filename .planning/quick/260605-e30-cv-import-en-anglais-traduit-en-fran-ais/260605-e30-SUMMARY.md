# Phase quick-260605-e30 Plan 01: CV Transcription Language-Neutral Summary

**One-liner:** DB migration removes 'français' constraint from cv.transcription system prompt so English CVs are transcribed in English, enabling correct language detection downstream.

## What Was Built

A single idempotent SQL migration that patches the `cv.transcription` LLM task row in `llm_tasks`. The `system_prompt_template` previously contained `Consigne : français, fidélité maximale au contenu` which forced the LLM to output French regardless of the source document language. The migration replaces this with `Consigne : fidélité maximale au contenu, dans la langue d'origine du document` — instructing faithful transcription in the document's original language.

## Root Cause

The extraction pipeline flows as:

1. `prepareCvText(pdfPath)` — LLM transcribes PDF via `cv.transcription` prompt → outputs `cvText`
2. `parallelExtractAndStream(cvText)` — calls `detectCvLanguage(cvText)` to detect language
3. All 4 branch prompts (`cv.branch.identity`, `cv.branch.experiences`, `cv.branch.education`, `cv.branch.skills`) receive `language_label` from detected language

With `Consigne : français` in step 1, the LLM always output French text. `detectCvLanguage` saw French and returned `'fr'`. All branches therefore extracted in French — even for English source CVs.
