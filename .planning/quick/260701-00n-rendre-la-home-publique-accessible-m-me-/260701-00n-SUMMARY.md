# Quick Task 260701-00n — Summary

**Description:** Rendre la home `/` réellement publique : accessible même quand l'utilisateur est connecté.
**Date:** 2026-07-01
**Status:** Complete

## Problem

The home route `/` was converted into a public marketing landing, but two things still forced authenticated users away from it:

1. `app/page.tsx` redirected logged-in users (`userId`) to `/dashboard`.
2. `components/authenticated-shell.tsx` always wrapped logged-in users in the full app shell (sidebar + header), so even without the redirect `/` would have rendered inside the dashboard layout instead of as the bare landing.

## Changes

### `app/page.tsx`
- Removed the `auth()` call and `redirect('/dashboard')` guard.
- `HomePage` is now a plain (non-async) component that always returns `<Landing />`.
- Removed now-unused `redirect` (`next/navigation`) and `auth` (`@clerk/nextjs/server`) imports.
- `metadata` export preserved.

### `components/authenticated-shell.tsx`
- Added an early `if (pathname === '/') return <>{children}</>;` right after the `isLoaded` guard, so the landing renders bare regardless of auth state (window-scroll preserved for GSAP ScrollTrigger).
- Removed the now-redundant `pathname === '/'` branch that lived inside the `!userId` block.

## Notes

- `middleware.ts` already lists `/` in `isPublicRoute`, so no middleware change was needed.
- Executor worktree isolation could not be used: the marketing-landing migration exists only as uncommitted working-tree state in the parent checkout, which git worktrees do not inherit. Edits were applied directly in the parent checkout.

## Verification

- `npx tsc --noEmit` → no errors in `app/page.tsx` or `components/authenticated-shell.tsx`.
