Create `lib/workflow/compute-step-status.ts` exporting:

```typescript
export type StepStatus = 'pending' | 'running' | 'done' | 'error';
```

For each workflow kind, export a function:

`computeXxxStepStates(input): { stepKey: string; status: StepStatus; errorMessage?: string }[]`

**Input must include:** latest `streamMeta` (typed), optional `partialData` for done detection where applicable, `isStreaming: boolean`, `errorStepKey: string | null` from hook, `persistedError: WorkflowLastError | null` from Supabase, and `workflowFailed: boolean` (server status `error`).

**Rules (D-05, D-07):**
- Multiple steps may be `running` if `activeBranches` has multiple entries.
- Steps before the active phase are `done` once past, or `pending` if not yet reached.
- On failure: steps matching `errorStepKey` or `persistedError.stepKey` → `error` with French message; others before are `done`, after remain `pending` unless parallel semantics say otherwise.
- **No percentage** progress bar.

Keep functions **pure** (no React). Unit-test optional — if no test file, skip.
